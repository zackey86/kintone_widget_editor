# バージョン管理ルール

このプロジェクトは[セマンティックバージョニング（Semantic Versioning）](https://semver.org/lang/ja/)に従ってバージョン管理を行います。

## バージョン形式

`MAJOR.MINOR.PATCH` の形式で、例: `v0.1.0`

- **MAJOR（メジャー）**: 後方互換性のない変更（破壊的変更）
- **MINOR（マイナー）**: 後方互換性を保った機能追加
- **PATCH（パッチ）**: 後方互換性を保ったバグ修正

## バージョンルール

### v0.x.x（開発初期段階）

- 初回リリースは `v0.1.0` から開始
- APIや機能が不安定な段階
- 後方互換性は保証されない

### v1.x.x（安定版）

- 初回の安定版リリースは `v1.0.0`
- 後方互換性を保証
- 破壊的変更は `MAJOR` バージョンを上げる

## バージョンアップの基準

### PATCH（0.0.1 → 0.0.2）

- バグ修正
- セキュリティパッチ
- ドキュメントの修正
- コードのリファクタリング（機能変更なし）

### MINOR（0.1.0 → 0.2.0）

- 新機能の追加
- 既存機能の改善（後方互換性あり）
- UI/UXの改善

### MAJOR（0.1.0 → 1.0.0 または 1.0.0 → 2.0.0）

- 破壊的変更
- APIの変更
- 既存機能の削除
- 動作の大幅な変更

## リリース手順

1. 変更内容をコミット
2. 適切なバージョン番号を決定
3. タグを作成: `git tag -a v0.1.0 -m "v0.1.0: 変更内容"`
4. タグをプッシュ: `git push origin v0.1.0`
5. GitHub CLIでリリース作成: `gh release create v0.1.0 --title "v0.1.0" --notes "変更内容"`
